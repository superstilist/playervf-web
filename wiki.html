Overview

PlayerVF is a cross-platform Flutter music and video player. It focuses on local media playback, playlists, favorites, lyrics, audio effects, YouTube Music tools, local sharing, metadata editing, cover art, and deep appearance customization.

PlatformsAndroid, Windows, and web are configured in the project.
Main purposePlay, organize, customize, download, and share music or video from one app.

User features

Local libraryScan folders, import music, remember library data, read metadata, and keep favorites and play stats.
PlaylistsCreate playlists, rename them, delete them, add songs, remove songs, and use system playlists like favorites or recent plays.
LyricsOpen plain or timed lyrics, search online, save selected lyrics, edit lyrics, and view them fullscreen.
MetadataEdit track info, pick cover art, and use MusicBrainz lookup to improve tags.

Playback

The app supports normal music controls, queues, shuffle, repeat, saved playback position, smooth transitions, volume control, and keyboard shortcuts. Video playback includes local video, YouTube video streams, quality switching, fullscreen mode, auto-hidden controls, and subtitle support.

Audio effects

  • Equalizer with presets and band controls.
  • Pitch, speed, and reverb controls.
  • Global or per-song effect settings.
  • Reset options for equalizer and all audio effects.

YouTube Music

PlayerVF can search YouTube Music, filter results, stream without downloading, download audio, download video, choose quality, download subtitles, cache stream URLs, and mark already downloaded items.

Desktop uses a Python helper, while Android uses native bridge/helper code for search, streaming, downloads, covers, and metadata.

Sharing

The app can start a local share server, share the current song or a full library, discover nearby share servers, fetch remote manifests, download remote tracks, pause or resume transfers, and avoid duplicate files.

Android also includes Wi-Fi Direct discovery, connection, disconnect, permissions, and trusted device support.

Customization

PlayerVF has theme modes, theme presets, accent color controls, RGB sliders, glass intensity, border radius, font size, card size, view mode, navigation position, particle effects, decoder choices, and performance modes.

ThemesMaterial, Graphite, Classic, Fox, Anime, Azure, Cosmic, Sunset, and Midnight.
EffectsSakura, snow, stars, bubbles, rain, or no particles.

Technical map

Main app

lib/main.dart starts the app, configures providers, applies theming, handles navigation, search, shortcuts, mini player, and player overlay behavior.

Core models

  • Music: track data, metadata, favorite state, stats, and paths.
  • Playlist: playlist name, song IDs, creation, and update times.
  • SettingsModel: theme, paths, layout, decoder, performance, video, and sharing settings.
  • LyricsDocument: parsed timed or plain lyrics.

Core services

  • MusicService: central playback, library, queue, lyrics, playlists, effects, video, and saved state.
  • MusicScannerService: scans local media and creates music objects.
  • ID3Parser: reads MP3, MP4, FLAC, APE, and fallback filename metadata.
  • YoutubeMusicService: bridges the app to Python or Android helpers.
  • LocalShareService: local network sharing and transfer handling.

How it works

Local scan flow

  1. User adds a music folder.
  2. The scanner finds audio and video files.
  3. Metadata and covers are read.
  4. Favorites, stats, and library snapshots are preserved.
  5. The UI updates with the refreshed library.

Playback flow

  1. User taps a track.
  2. The service prepares the player and opens the media.
  3. Effects and decoder settings are applied.
  4. Playback starts and OS media controls update.

Sharing flow

  1. One device starts sharing.
  2. Another device discovers it.
  3. The receiver fetches a manifest and chooses tracks.
  4. Files transfer with progress and duplicate checks.
